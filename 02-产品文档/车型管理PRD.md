## 1. 业务背景与使用角色

### 1.1 业务背景

车型管理模块是车辆管理系统中系统管理的核心基础数据模块，负责对系统中涉及的所有车型进行统一管理。车型数据是车辆建档、车辆资产管理、车辆调拨等业务模块的数据来源基础，为各业务模块提供统一、规范的车型信息引用。

车型管理的核心特点：
- **一车型多版本**：同一车型可存在多个版本，各版本在底盘、动力、参数等方面存在差异
- **全局共享**：车型数据为全系统共享的基础数据，无项目归属隔离
- **版本追溯**：支持对车型的修改操作进行变更日志记录，实现数据可溯源

本模块与关联模块的关系：
- 车辆建档模块：引用车型数据，选择车型名称+版本后自动填充关联字段
- 车辆资产管理模块：引用车型数据展示车辆所属车型信息

### 1.2 使用角色

| 角色       | 车型管理模块中的职责                     | 客户端   |
| -------- | ------------------------------- | ----- |
| 管理员      | 查看全部车型、新建/编辑/删除车型、新建版本、导出数据 | PC    |
| 项目经理     | 查看全部车型、新建/编辑/删除车型、新建版本       | PC    |
| 财务人员     | 查看全部车型                          | PC    |
| 管理层      | 查看全部车型                          | PC    |
| 车队队长/司机  | 查看全部车型（如系统管理入口开放）              | APP   |

### 1.3 核心目标与价值

| 目标维度   | 说明                                |
| ------ | --------------------------------- |
| 数据标准化  | 统一管理车型数据，避免各业务模块车型信息不一致        |
| 版本管理   | 支持同一车型多版本管理，适应车辆迭代升级场景         |
| 数据溯源   | 变更日志记录所有修改行为，支持数据变更追溯           |
| 操作便捷   | 可折叠行展示车型与版本关系，一目了然              |
| 数据复用   | 为车辆建档、资产管理等模块提供统一车型数据来源        |

## 3. 功能详细设计

### 3.1 F01 车型列表页

#### 3.1.1 页面定位

车型管理的统一入口页面，以可折叠行的形式展示车型及版本层级关系，支持多维度查询和批量导出。

#### 3.1.2 页面结构

页面自上而下分为三个区域：
1. **查询筛选区**：提供查询条件输入与操作按钮
2. **数据列表区**：以可折叠表格形式展示车型数据
3. **分页区**：展示分页控件与数据统计

#### 3.1.3 查询筛选区

查询条件与查询按钮放置在同一行显示。

| 元素     | 类型       | 配置说明                                                  |
| ------ | -------- | ----------------------------------------------------- |
| 车型名称（中）| 文本输入框    | 默认显示，支持模糊搜索                                           |
| 车型名称（英）| 文本输入框    | 默认显示，支持模糊搜索                                           |
| 车型版本   | 文本输入框    | 默认显示，支持模糊搜索                                           |
| 车型别名   | 下拉单选     | 默认显示，枚举值：扫路车、运输车、小型清洗机、小型扫路机、皮卡等                   |
| 车型编码   | 文本输入框    | 收起，展开后显示，支持模糊搜索                                       |
| 车型来源   | 下拉单选     | 收起，展开后显示，枚举值：自研、非自研                                 |
| 动力类别   | 下拉单选     | 收起，展开后显示，枚举值：纯电动、柴油、汽油、混动、天然气                        |
| 排放标准   | 下拉单选     | 收起，展开后显示，枚举值：国三、国四、国五、国六、纯电动、非道路移动机械                 |
| 整车厂商   | 文本输入框    | 收起，展开后显示，支持模糊搜索                                       |
| 创建人    | 文本输入框    | 收起，展开后显示，支持模糊搜索                                       |
| 创建时间   | 日期范围选择器  | 收起，展开后显示                                               |
| 查询按钮   | 主按钮      | 点击执行查询，列表刷新为查询结果                                      |
| 重置按钮   | 次按钮      | 点击重置所有查询条件至默认状态（所有条件为空）                              |
| 展开/收起  | 文字链接     | 默认仅显示前4个查询条件，点击展开显示所有查询条件，再次点击收起                     |
| 新建按钮   | 主按钮      | 点击打开新建车型信息页面；仅管理员、项目经理可见                            |
| 导出按钮   | 次按钮      | 点击执行导出；仅管理员可见                                         |

**查询条件显示规则：**
- 默认显示前4个查询条件：车型名称（中）、车型名称（英）、车型版本、车型别名
- 其余查询条件收起，点击"展开"后显示全部查询条件
- 查询条件与查询按钮、重置按钮在同一行排列

#### 3.1.4 数据列表区

列表采用**可折叠行**设计，外层行为车型维度信息，内层行为版本维度信息。

**外层行（车型行）显示字段：**

| 序号 | 字段名      | 字段标识           | 宽度建议   | 对齐方式 | 说明              |
| -- | -------- | -------------- | ------ | ---- | --------------- |
| 0  | 展开图标     | expand_icon    | 50px   | 居中   | 点击展开/收起该车型的版本行  |
| 1  | 车型名称（中）  | model_name     | 180px  | 左对齐  |                 |
| 2  | 车型名称（英）  | model_name_en  | 180px  | 左对齐  |                 |
| 3  | 车型别名     | model_alias    | 140px  | 左对齐  | 枚举值展示           |
| 4  | 车型编码     | model_code     | 140px  | 左对齐  |                 |
| 5  | 车型来源     | model_source   | 120px  | 左对齐  | 自研/非自研          |
| 6  | 创建人      | creator        | 120px  | 左对齐  |                 |
| 7  | 创建时间     | create_time    | 180px  | 左对齐  | 格式：YYYY-MM-DD HH:mm:ss |

**内层行（版本行，展开后显示）显示字段：**

| 序号 | 字段名      | 字段标识           | 宽度建议   | 对齐方式 | 说明              |
| -- | -------- | -------------- | ------ | ---- | --------------- |
| 0  | 复选框      | checkbox       | 50px   | 居中   | 支持全选/单选，用于导出操作   |
| 1  | 车型版本     | model_version  | 140px  | 左对齐  |                 |
| 2  | 车辆型号     | vehicle_model  | 160px  | 左对齐  |                 |
| 3  | 动力类别     | power_type     | 120px  | 左对齐  | 枚举值展示           |
| 4  | 排放标准     | emission_std   | 140px  | 左对齐  | 枚举值展示           |
| 5  | 整车厂商     | manufacturer   | 160px  | 左对齐  |                 |
| 6  | 操作       | actions        | 200px  | 居中   | 详情、编辑、新建版本、删除   |

**折叠行交互规则：**
1. 默认所有车型行处于**收起**状态
2. 点击外层行的展开图标（或行任意区域），展开该车型对应的所有版本行
3. 展开状态下，再次点击收起
4. 按照创建时间由近到远倒序排列（车型行级别排序）

**操作列按钮（版本行）：**

| 操作   | 触发行为                | 可见角色                  |
| ---- | ------------------- | --------------------- |
| 详情   | 打开该车型版本的车型信息详情页面（查看态） | 所有角色                  |
| 编辑   | 打开该车型版本的车型信息详情页面（编辑态） | 管理员、项目经理              |
| 新建版本 | 打开新建版本页面，自动填充该车型相关信息   | 管理员、项目经理              |
| 删除   | 弹出二次确认弹窗后删除该版本         | 管理员、项目经理              |

**行交互：** 行hover高亮。

**复选框交互：**
- 内层版本行每行存在复选框，支持单行勾选
- 表头存在全选复选框，勾选后选中当前页所有版本行
- 勾选状态用于导出操作

#### 3.1.5 分页区

| 配置项     | 规则                                |
| ------- | --------------------------------- |
| 每页条数选项  | 10、20、50、100，默认10                |
| 显示信息    | 总记录数、当前页码/总页数                    |
| 翻页控件    | 首页、上一页、页码、下一页、末页                 |
| 切换每页条数  | 下拉选择，切换后从第一页重新加载                 |

**说明：** 分页作用于外层车型行。每页显示N个车型（含其版本行）。

### 3.2 F02 车型信息详情页

#### 3.2.1 页面定位

展示某一车型版本的完整字段信息，支持查看态与编辑态切换，底部展示变更日志模块，实现数据溯源。

#### 3.2.2 页面结构

页面自上而下分为：
1. **顶部操作栏**：包含"关闭"、"编辑"按钮
2. **信息内容区**：按分组展示车型及版本字段信息
3. **变更日志区**：展示该车型版本的操作变更记录
4. **底部操作栏**（编辑态）：包含"确定"和"取消"按钮，固定在页面底部

#### 3.2.3 信息分组与字段展示

信息按以下4个分组展示，各分组使用折叠面板或分组标题分隔：

| 序号 | 分组名称   | 说明                       |
| -- | ------ | ------------------------ |
| 1  | 车型信息   | 车型来源、名称、别名、用途分类、编码、编码头等 |
| 2  | 版本信息   | 车型版本、车辆型号、动力类别、排放标准等     |
| 3  | 主要参数信息 | 底盘、整车、发动机、电机、清扫/冲洗宽度等    |
| 4  | 其他信息   | 备注、附件                    |

**查看态**：所有字段只读展示，附件类字段支持点击预览/下载。

**编辑态**：字段按"是否支持编辑"规则变为可输入状态，必填项标红色星号。

#### 3.2.4 顶部操作栏

| 按钮     | 类型   | 可见角色             | 行为                                    |
| ------ | ---- | ---------------- | ------------------------------------- |
| 关闭     | 次按钮  | 所有角色             | 关闭当前详情页面，返回列表页                        |
| 编辑     | 主按钮  | 管理员、项目经理         | 进入编辑态，底部显示固定操作栏（确定/取消）               |
| 新建版本   | 主按钮  | 管理员、项目经理         | 打开新建版本页面，自动填充该车型相关信息                 |

#### 3.2.5 变更日志模块

位于信息内容区下方，以列表形式展示该车型版本的所有操作变更记录。

**修改行为记录字段：**

| 操作时间   | 删除操作的时间          |

**列表显示：**

| 序号 | 字段名   | 宽度建议   | 对齐方式 | 说明         |

### 3.3 F03 车型信息编辑

#### 3.3.1 触发方式

1. 用户在列表页版本行点击"编辑"按钮
2. 用户在详情页点击"编辑"按钮

#### 3.3.2 编辑态规则

1. 点击"编辑"后，信息内容区所有支持编辑的字段变为可输入状态
2. 编辑态下，底部显示固定操作栏，包含"确定"和"取消"按钮，按钮右侧对齐
3. 必填项标红色星号
4. **不可编辑字段**（车型来源、编码头）保持只读状态，置灰显示
5. 点击"确定"：
   - 系统执行表单校验（必填项、唯一性校验等）
   - 校验成功 → 保存数据，记录变更日志（记录修改字段、修改前值、修改后值），退出编辑态，显示"保存成功"Toast提示
   - 校验失败 → 在对应字段下方显示红色错误提示，编辑态不退出
6. 点击"取消" → 退出编辑态，不保存任何修改，恢复至修改前的数据
7. 车型名称（中）编辑时需保持全局唯一
8. 车型编码编辑时需保持全局唯一（如有值）

## 4. 业务规则汇总

### 4.1 车型唯一性规则

1. **车型名称（中）全局唯一**：系统中不可存在重复的中文车型名称
2. **车型名称（英）全局唯一**：如填写，系统中不可存在重复的英文车型名称
3. **车型编码全局唯一**：如填写，系统中不可存在重复的车型编码

### 4.2 版本唯一性规则

1. **同一车型下版本号不可重复**：同一车型名称（中）下，车型版本字段值不可重复
2. **已被车辆引用的版本不可删除**：删除前需校验该版本是否被车辆建档数据引用

### 4.3 删除规则

1. 仅支持删除未被车辆引用的车型版本
2. 删除前需二次确认
3. 删除后记录删除操作日志

### 4.4 载重分类计算规则

| 最大重量范围            | 载重分类  |
| ------------------ | ----- |
| 最大重量 ≥ 8000 KG     | 重卡    |
| 最大重量 ≤ 3000 KG     | 微卡    |
| 3000 KG < 最大重量 < 8000 KG | 轻卡    |

**说明：** 最大重量为空时，载重分类显示为空。

### 4.5 字段联动规则

| 触发字段   | 触发条件       | 联动行为                        |
| ------ | ---------- | --------------------------- |
| 车型来源   | 选择"非自研"   | 编码头下拉选项默认选中5100（如5100在选项中） |
| 最大重量   | 填写数值后      | 载重分类字段自动计算并显示                |

### 4.6 变更日志记录规则

1. 系统需要对用户对车型的修改和删除操作进行记录
2. 修改行为记录：操作对象、操作类型、修改字段、修改前、修改后、操作人、操作时间、操作来源
3. 删除行为记录：操作对象、操作类型、操作人、操作时间、操作来源
4. 变更日志按操作时间由近到远倒序排列
