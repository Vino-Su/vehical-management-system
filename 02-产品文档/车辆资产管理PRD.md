## 1. 业务背景与使用角色

### 1.1 业务背景

车辆资产管理模块是车辆管理系统中针对已完成资产入池车辆的核心管理模块。在车辆建档模块完成车辆信息录入后，车辆数据流转至资产管理模块，形成统一的车辆资产台账。本模块为用户提供了查看、编辑、分配、导出车辆资产数据的能力，是车辆全生命周期管理中"资产管理→调拨→运行→退出"环节的关键起点。

本模块与车辆建档模块的关系：
- 车辆建档模块负责车辆数据的**录入入口**，是数据源头
- 车辆资产管理模块负责车辆资产的**查看与管理**，是建档后车辆数据的运营管理载体

### 1.2 使用角色

| 角色       | 资产管理模块中的职责                     | 客户端   |
| -------- | ----------------------------- | ----- |
| 管理员      | 查看所有资产、编辑资产信息、分配车辆、变更车辆状态、删除车辆、导出数据 | PC+APP |
| 项目经理     | 查看本项目资产、编辑资产信息、分配车辆、变更车辆状态、删除车辆、导出数据 | PC+APP |
| 财务人员     | 查看所有资产信息、导出数据                | PC    |
| 管理层      | 查看车辆资产信息                     | PC    |
| 车队队长/司机  | 查看自身项目的车辆资产信息                | APP   |

### 1.3 核心目标与价值

| 目标维度    | 说明                              |
| ------- | ------------------------------- |
| 资产台账    | 形成统一的车辆资产台账，一览车辆资产全貌            |
| 资产维护    | 支持对资产信息进行编辑更新，保持数据时效性           |
| 车辆分配    | 支持将待分配车辆分配至指定项目，实现资产流转          |
| 车辆状态变更  | 支持批量变更车辆状态，实现车辆状态的及时更新与管理       |
| 全生命周期档案 | 详情页承载车辆全生命周期数据（基础信息、调度信息、变更日志等） |
| 数据导出    | 支持选中导出或全量导出，满足报表和审计需求           |

## 3. 功能详细设计

#### 3.1.1 页面定位

车辆资产管理的核心入口页面，以列表形式展示所有已入池车辆的资产数据，支持多维度查询和批量操作。

1. **查询筛选区**：提供查询条件输入与操作按钮
2. **数据列表区**：以表格形式展示车辆资产数据
3. **分页区**：展示分页控件与数据统计

### 3.2 F02 车辆资产详情页

#### 3.2.1 页面定位

展示车辆的全生命周期数据，以Tab标签形式组织不同维度的信息。用户可查看、编辑车辆资产的完整档案。

#### 3.2.2 页面结构

页面自上而下分为：
1. **顶部操作栏**：包含"关闭"按钮和"编辑"按钮
2. **Tab标签区**：展示各维度信息标签
3. **内容区**：展示当前选中Tab的详细内容
4. **底部操作栏**（编辑态）：包含"确定"和"取消"按钮，固定在页面底部

#### 3.2.3 Tab标签定义

| 序号 | Tab名称  | 本期建设 | 显示条件             | 说明                              |
| -- | ------ | ---- | ---------------- | ------------------------------- |
| 1  | 基础信息   | 是    | 始终显示             | 默认显示的Tab，展示车辆全维度基础字段            |
| 2  | 调度信息   | 是    | 始终显示             | 展示该车辆的调度记录列表                   |
| 3  | BOM清单  | 是    | 仅当车辆"资产来源"为"自研"时显示 | 展示该车辆的BOM物料清单及零部件变更记录          |
| 4  | 维保信息   | 否    | 始终显示（置灰）         | 本期不建设，Tab标签置灰显示，标注"暂未开放"        |
| 5  | 合规信息   | 否    | 始终显示（置灰）         | 本期不建设，Tab标签置灰显示，标注"暂未开放"        |
| 6  | 风险信息   | 否    | 始终显示（置灰）         | 本期不建设，Tab标签置灰显示，标注"暂未开放"        |
| 7  | 费用信息   | 否    | 始终显示（置灰）         | 本期不建设，Tab标签置灰显示，标注"暂未开放"        |
| 8  | 变更日志   | 是    | 始终显示             | 展示该车辆的操作变更记录                   |

**默认显示：** 基础信息Tab。

**BOM清单Tab动态显示规则：**
- 当车辆的"资产来源"字段值为"自研"时，在调度信息Tab与维保信息Tab之间动态插入显示"BOM清单"Tab
- 当车辆的"资产来源"字段值为其他值（外购、租赁、融资租赁、借用）时，不显示"BOM清单"Tab，不占位
- 判断依据为车辆建档中的"资产来源"字段，非车型管理中的"车型来源"字段

#### 3.2.4 顶部操作栏

| 按钮   | 类型   | 可见角色             | 行为                                |
| ---- | ---- | ---------------- | --------------------------------- |
| 关闭   | 次按钮  | 所有角色             | 关闭当前详情页面，返回列表页                    |
| 编辑   | 主按钮  | 管理员、项目经理         | 进入当前Tab页面的编辑状态，底部显示固定操作栏（确定/取消）；BOM清单Tab为只读，切换到该Tab时隐藏编辑按钮 |

#### 3.2.5 编辑态规则

1. 点击"编辑"后，仅当前Tab页面进入编辑态，其他Tab页面不受影响
2. 编辑态下，底部显示固定操作栏，包含"确定"和"取消"按钮，按钮右侧对齐
3. 点击"确定"：执行当前Tab页面内容校验，校验成功则保存并记录变更日志，校验失败则显示错误提示
4. 点击"取消"：退出编辑态，不保存任何修改
5. 编辑保存前需要对页面内容进行校验，校验失败则保存失败
6. 保存成功后，自动记录变更日志（修改字段、修改前后值、操作人、操作时间等）

### 3.3 F03 基础信息Tab

#### 3.3.1 页面定位

展示车辆的完整基础档案信息，涵盖车辆从车型到硬件配置的全维度数据。查看态为只读展示，编辑态可修改字段。

基础信息Tab围绕车辆的车型、车辆等基础信息展开，分为以下16个信息分组，各分组使用折叠面板或分组标题分隔：

| 序号 | 分组名称   | 说明                       |
| -- | ------ | ------------------------ |
| 1  | 车辆基础信息 | 车架号、车牌号等车辆标识信息           |
| 2  | 归属信息   | 项目归属、资产来源等归属信息           |
| 3  | 车辆照片   | 车辆外观照片                   |
| 4  | 行驶证信息  | 行驶证照片及注册信息               |
| 5  | 车型信息   | 车型名称、版本及关联自动获取的车型参数      |
| 6  | 整车信息   | 整车铭牌、制造日期等               |
| 7  | 底盘信息   | 底盘铭牌、轮胎型号等               |
| 8  | 主发动机信息 | 主发动机铭牌照片                 |
| 9  | 副发动机信息 | 副发动机铭牌、厂家、型号、功率等         |
| 10 | 变速箱信息  | 变速箱铭牌、厂家、型号等             |
| 11 | 上装信息   | 风机、水泵等上装设备信息             |
| 12 | 其他     | 其他配置附件、备注                |
| 13 | 硬件信息   | 硬件设备列表，可动态增减行            |
| 14 | 软件信息   | 软件信息列表，可动态增减行            |
| 15 | 摄像头信息  | 摄像头参数列表，可动态增减行           |

#### 3.3.3 字段定义

各分组字段定义与车辆建档模块中"新增车辆信息"的字段一致（详见车辆建档PRD 3.2.3节），此处不再重复罗列。

**与建档模块字段的关系说明：**
- 查看态：所有字段只读展示，附件类字段支持点击预览/下载
- 编辑态：字段可编辑，校验规则与建档模块一致
- 车型信息分组：车型名称、车型版本为下拉选择，确定后自动填充关联字段（只读）；其余车型信息字段不可手动编辑

#### 3.3.4 编辑态交互流程

1. 用户在详情页点击"编辑"→ 基础信息Tab进入编辑态
2. 可编辑字段变为可输入状态，必填项标红色星号
3. 车型名称选择后，车型版本下拉选项自动填充；选择车型版本后，其余车型信息字段自动填充（只读）
4. 硬件信息、软件信息、摄像头信息分组支持"添加行"和"删除行"操作
5. 点击"确定"：
   - 系统执行表单校验（必填项、格式校验等）
   - 校验成功 → 保存数据，记录变更日志（记录修改字段、修改前值、修改后值），退出编辑态，显示"保存成功"Toast提示
   - 校验失败 → 在对应字段下方显示红色错误提示，编辑态不退出
6. 点击"取消" → 退出编辑态，不保存任何修改，恢复至修改前的数据

### 3.6 F09 BOM清单Tab

#### 3.6.1 页面定位

展示自研车辆的BOM（Bill of Materials）物料清单信息，包含出厂原始BOM数据及售后维修产生的零部件变更记录。帮助用户了解车辆当前的零部件组成及历史变更情况，实现零部件级别的物料追溯。

本Tab为动态Tab，仅当车辆的"资产来源"为"自研"时显示，其他车辆不显示该Tab。数据来源于两个外部系统：

| 数据来源  | 同步方式  | 触发时机    | 数据性质           |
| ----- | ----- | ------- | -------------- |
| MES系统 | 一次性同步 | 车辆生产完毕后 | 出厂BOM静态快照，不再变更 |
| 售后系统  | 增量推送  | 每次维修完成时 | 零部件变更流水，持续追加   |

#### 3.6.2 页面结构

页面自上而下分为：
1. **概览统计条**：展示BOM数据的关键统计信息
2. **视图切换区**：切换"当前BOM"和"变更记录"两个视图
3. **搜索筛选区**：根据当前视图提供对应的查询条件
4. **操作区**：导出、刷新等操作按钮
5. **数据列表区**：以表格形式展示BOM数据或变更记录
6. **分页区**：展示分页控件与数据统计

#### 3.6.3 概览统计条

在BOM清单Tab顶部展示轻量统计条，帮助用户快速感知车辆零部件变化情况。

**显示信息：**

| 元素       | 类型   | 说明                                       |
| -------- | ---- | ---------------------------------------- |
| 总件数      | 数值   | 当前BOM中的零部件总条目数                           |
| 已变更件数    | 数值+链接 | 售后系统中发生变更的零部件数量，可点击跳转至"当前BOM"视图并自动筛选为"仅已变更" |
| 出厂BOM同步时间 | 时间文本 | MES系统同步出厂BOM的时间，格式：YYYY-MM-DD HH:mm      |

**显示规则：**
- 出厂BOM同步时间后标注"出厂基准"标识
- 当BOM数据为空时（MES尚未同步或同步失败），统计条中总件数显示为"—"，已变更件数显示为"—"，同步时间显示为"尚未同步"

**示例：**
```
📦 总件数 1,286    🔄 已变更 12（点击查看）    🕐 出厂BOM同步时间 2025-03-15 14:30（出厂基准）
```

#### 3.6.4 视图切换区

提供"当前BOM"和"变更记录"两个视图的切换控件，以Tab子标签或按钮组形式展示。

| 视图名称   | 默认选中 | 定位                           |
| ------ | ---- | ---------------------------- |
| 当前BOM  | 是    | 展示车辆**当前最新**的零部件清单，回答"车上现在装的什么件" |
| 变更记录   | 否    | 展示车辆**历史**零部件变更流水，回答"哪些件被换过、为什么换" |

##### 3.6.5.1 视图说明

展示车辆当前的零部件完整清单。数据以MES出厂BOM为底表，将售后变更按件号覆盖到对应行上，形成反映当前真实状态的BOM。

##### 3.6.5.2 搜索筛选区

| 级别      | 下拉单选     | 枚举值：全部、L1（总成级）、L2（分总成级）、L3（零件级），选项由数据动态生成      |
| 变更状态    | 下拉单选     | 枚举值：全部、仅已变更、仅未变更                            |
| 查询按钮    | 主按钮      | 点击执行查询，列表刷新为查询结果                            |

**表格交互规则：**
- 表格支持横向滚动，当屏幕宽度不足以显示所有列时，固定前3列（级别、件号、名字），其余列可横向滚动
- 默认显示列：级别、件号、名字、规格型号、用量、单位、供应商、物料分类、变更（共9列）
- 其余列（材料、采购选项、零件分类、追溯分类、零件制造策略、生产车间）收起，通过表格右上角"列设置"按钮可自定义显示列
- 行hover高亮

**排序规则：** 默认按级别+件号排序，保持MES系统的原始BOM层级结构顺序。

**"级别"字段缩进规则：**
- L1级（总成级）：不缩进，正常显示
- L2级（分总成级）：缩进2个中文字符宽度
- L3级（零件级）：缩进4个中文字符宽度
- 如存在更深层级，依此规律递增缩进

##### 3.6.5.5 零部件变更时间线弹窗

用户点击"变更"列的🔄图标后，弹出零部件变更时间线弹窗，以弹窗形式展示该零件的完整变更历史。

**弹窗规格：**

| 配置项   | 值                               |
| ----- | ------------------------------- |
| 弹窗标题  | "{件号} {名字} · 变更记录"，如"A001-01 驱动电机 · 变更记录" |
| 弹窗宽度  | 640px                           |
| 弹窗高度  | 自适应内容，最大高度不超过屏幕高度的70%           |
| 弹窗位置  | 页面水平垂直居中                        |
| 遮罩层   | 半透明黑色遮罩，点击遮罩不关闭弹窗               |
| 底部操作栏 | 仅"关闭"按钮，居右显示                    |

**弹窗内容 — 变更时间线：**

弹窗内以时间线形式展示该零件的所有变更记录，按时间倒序排列（最近的变更在最上方）。时间线由多条记录节点组成，节点之间以竖线连接。

**每条变更记录节点的展示字段：**

| 字段名       | 显示位置 | 说明                                                   |
| ---------- | ------ | ---------------------------------------------------- |
| 变更时间      | 节点头部  | 格式：YYYY-MM-DD                                       |
| 变更来源标签    | 节点头部  | 标签样式：出厂装机（灰色标签）/ 售后更换（蓝色标签）                           |
| 变更前件号     | 变更前后对比区 | 变更前的件号；出厂装机记录此栏显示"—"                                  |
| 变更后件号     | 变更前后对比区 | 变更后的件号；出厂装机记录显示初始件号                                   |
| 变更前供应商    | 变更前后对比区 | 变更前的供应商；出厂装机记录此栏显示"—"                                  |
| 变更后供应商    | 变更前后对比区 | 变更后的供应商；出厂装机记录显示初始供应商                                  |
| 变更前规格型号   | 变更前后对比区 | 变更前的规格型号；出厂装机记录此栏显示"—"                                 |
| 变更后规格型号   | 变更前后对比区 | 变更后的规格型号；出厂装机记录显示初始规格型号                                |
| 工单号       | 节点底部  | 售后维修工单编号，蓝色链接样式，点击可跳转至售后系统查看工单详情；出厂装机记录此栏不显示           |
| 变更原因      | 节点底部  | 售后维修的更换原因描述；出厂装机记录此栏不显示                                |

**变更前后对比区设计：**

变更前后信息以左右对比或上下对比的形式展示，变更前信息在前（灰色文字），箭头→连接，变更后信息在后（黑色加粗文字），突出变化。

**变更类型为"更换"的节点示例：**

```
● 2026-05-18  [售后更换]
  件号:    CATL-200    →  CATL-200B
  供应商:  宁德时代      →  宁德时代
  规格:    380V/200Ah  →  380V/200Ah(改进型)
  工单号:  WX202605180032
  变更原因: 电机异响
```

**变更类型为"出厂装机"的节点示例：**

```
● 2025-03-15  [出厂装机]
  件号:    CATL-200
  供应商:  宁德时代
  规格:    380V/200Ah
```

**交互规则：**
- 点击🔄图标打开弹窗，同一时间仅打开一个变更时间线弹窗
- 点击弹窗底部"关闭"按钮关闭弹窗
- 点击弹窗右上角×关闭弹窗
- 按 ESC 键关闭弹窗
- 弹窗内容超出最大高度时，弹窗内支持纵向滚动
- 出厂装机记录始终作为时间线的最后一条节点（时间最早）
- 当该零件仅有出厂装机记录、无售后变更时，弹窗中仅显示一条出厂装机节点，并在弹窗标题下方显示提示："该零部件尚未发生售后变更"

| 件号       | part_no                | 文本     | 零部件唯一编号               |
| 名字       | part_name              | 文本     | 零部件名称                 |
| 材料       | material               | 文本     | 材质信息                  |
| 规格型号     | specification          | 文本     | 规格型号描述                |
| 用量       | quantity               | 数值     | 数量                    |
| 单位       | unit                   | 文本     | 计量单位                  |
| 供应商      | supplier               | 文本     | 供应商名称                 |
| 物料分类     | material_category      | 文本     | 物料分类                  |
| 采购选项     | purchase_option        | 文本     | 采购选项                  |
| 零件分类     | part_category          | 文本     | 零件分类                  |

| 变更类型     | change_type       | 枚举     | 更换、新增、移除                           |
| 变更前件号    | before_part_no    | 文本     | 变更前的件号；新增类型为空                      |
| 变更前供应商   | before_supplier   | 文本     | 变更前的供应商                            |
| 变更前规格型号  | before_spec       | 文本     | 变更前的规格型号                           |
| 变更后件号    | after_part_no     | 文本     | 变更后的件号；移除类型为空                      |
| 变更后供应商   | after_supplier    | 文本     | 变更后的供应商                            |
| 变更后规格型号  | after_spec        | 文本     | 变更后的规格型号                           |
| 工单号      | work_order_no     | 文本     | 售后维修工单编号，关联售后系统                    |
| 变更原因     | change_reason     | 文本     | 售后维修的更换原因                          |
| 变更时间     | change_time       | 日期时间   | 零部件变更发生的时间                         |
| 数据来源     | data_source       | 文本     | 固定为"售后系统"                          |

#### 3.6.9 数据同步规则

##### 3.6.9.1 MES系统数据同步

| 规则项      | 说明                                          |
| -------- | ------------------------------------------- |
| 同步触发时机   | 车辆生产完毕后，由MES系统主动推送或本系统拉取                     |
| 同步方式     | 一次性全量同步                                     |
| 同步内容     | 该车架号对应的完整出厂BOM清单                             |
| 同步后状态    | 出厂BOM数据锁定，不再更新                              |
| 同步失败处理   | 标记同步状态为"同步异常"，在BOM清单Tab中展示错误提示；支持管理员手动触发重新同步 |
| 数据归属     | 出厂BOM表（vehicle_bom_original）                 |

##### 3.6.9.2 售后系统数据同步

| 规则项      | 说明                                          |
| -------- | ------------------------------------------- |
| 同步触发时机   | 每次维修完成时，由售后系统主动推送变更记录                        |
| 同步方式     | 增量推送，仅推送本次变更涉及的零部件信息                          |
| 同步内容     | 变更零部件的件号、变更类型、变更前后信息、工单号、变更原因、变更时间            |
| 同步频率     | 实时推送（推荐）或定时拉取（最长间隔不超过24小时）                    |
| 同步失败处理   | 标记该条记录为"待重试"，系统自动重试3次；3次失败后标记为"同步异常"，通知管理员    |
| 数据归属     | BOM变更记录表（vehicle_bom_change）                 |

##### 3.6.9.3 当前BOM视图的数据合并逻辑

当前BOM视图中展示的数据为"计算后的结果"，合并逻辑如下：

1. 以出厂BOM表数据为基础数据集
2. 按件号匹配BOM变更记录表中的数据，对同一件号存在多条变更记录的，取最新一条变更后的信息
3. 变更类型为"更换"：覆盖该件号的供应商、规格型号等字段为变更后的值，并在"变更"列标记🔄图标
4. 变更类型为"新增"：在BOM中新增一条零部件记录
5. 变更类型为"移除"：从当前BOM中移除该零部件记录（出厂BOM原始数据保留，仅展示时不显示）
6. 每次加载当前BOM视图时，实时执行合并计算

#### 3.6.10 编辑态规则

BOM清单Tab为**只读**，不进入编辑态。具体规则：

1. 切换到BOM清单Tab时，页面顶部"编辑"按钮隐藏
2. BOM清单Tab不支持任何编辑、新增、删除操作
3. 所有数据变更由MES系统和售后系统通过接口同步，保持数据源唯一性
4. 如需修正BOM数据错误，需在数据源系统（MES或售后系统）中操作

#### 3.6.11 与变更日志Tab的关系

BOM清单Tab中的"变更记录"视图与变更日志Tab（3.5节）记录的内容不同，两者独立维护，不做合并。

| 维度     | 变更日志Tab               | BOM清单Tab - 变更记录视图      |
| ------ | ---------------------- | ---------------------- |
| 数据来源   | 用户在车辆管理系统中的操作行为        | MES系统/售后系统推送的外部数据      |
| 记录对象   | 车辆属性字段（如车辆状态、归属项目等）     | 零部件实体（如驱动电机、传感器等）       |
| 记录粒度   | 字段级                    | 件号级                    |
| 主要用途   | 操作审计                   | 物料追溯                   |
| 是否可编辑  | 不可编辑，仅记录               | 不可编辑，仅展示               |

**联动说明：** 当售后系统推送BOM变更数据时，系统自动在变更日志Tab中记录一条摘要日志：

| 日志字段   | 填写规则                                           |
| ------ | ---------------------------------------------- |
| 操作类型   | BOM变更                                          |
| 修改字段   | 格式："零部件变更({件号} {名字})"，如"零部件变更(A001-01 驱动电机)"    |
| 修改前    | 变更前的件号 + 规格，如"CATL-200 380V/200Ah"              |
| 修改后    | 变更后的件号 + 规格，如"CATL-200B 380V/200Ah(改进型)"        |
| 操作人    | 系统同步                                           |
| 操作时间   | 数据同步时间                                          |
| 操作来源   | 售后系统                                           |

## 4. 业务规则汇总

### 4.1 编辑规则

1. 车辆资产详情页在进行编辑时，编辑仅对当前Tab页生效
2. 点击了某个Tab页后进行编辑，仅编辑当前Tab页内容
3. 提交后仅保存当前Tab页内容
4. 保存前需对页面内容进行校验，校验失败则保存失败
5. 保存成功后自动记录变更日志
6. BOM清单Tab为只读，不支持编辑。切换到该Tab时隐藏编辑按钮

### 4.2 操作日志记录规则

1. 系统需要对用户对车辆的操作行为进行记录，包括修改和删除
2. 修改行为记录：操作对象、操作类型、修改字段、修改前、修改后、操作人、操作时间、操作来源，并在"变更日志"Tab中体现
3. 删除行为记录：操作对象、操作类型、操作人、操作时间、操作来源
4. 车辆状态变更行为记录：操作对象、操作类型（状态变更）、修改字段（车辆状态）、修改前（原状态）、修改后（新状态）、操作人、操作时间、操作来源
5. 变更日志按操作时间倒序排列
6. BOM变更数据由外部系统（售后系统）推送同步时，自动在变更日志Tab中记录一条BOM变更摘要日志

### 4.3 车辆分配规则

1. 仅车辆状态为"待分配"的车辆支持分配操作
2. 已分配的车辆需走车辆调拨流程（非本模块处理）
3. 分配操作变更车辆的"归属项目"字段
4. 分配成功后车辆状态由"待分配"变更为"在库-正常使用"

### 4.4 车辆状态变更规则

1. 车辆状态变更仅修改"车辆状态"字段，不涉及其他字段变更
2. "待分配"状态不可通过状态变更弹窗手动设置，仅通过车辆分配流程自动变更为"在库-正常使用"
3. 状态变更后，系统自动在变更日志Tab中记录变更信息（修改字段：车辆状态，修改前：原状态，修改后：新状态）
4. 已报废、已处置状态的车辆不建议再次变更状态，如需变更需走特殊审批流程（本期暂不限制，仅做提示）
5. 状态变更操作权限：仅管理员、项目经理可操作

### 4.5 车辆状态枚举

| 枚举值            | 说明          |
| -------------- | ----------- |
| 在库-闲置留用     | 闲置但保留使用     |
| 在库-闲置可调出         | 闲置可调配至其他项目  |
| 在库-维修  | 车辆维修中       |
| 在库-保养中          | 车辆保养中     |
| 在库-正常使用             | 车辆在库，车辆正常使用中      |
| 在库-待报废             | 车辆待提出报废申请      |
| 待分配            | 归属项目为空，待分配  |
| 已报废            | 报废申请审批通过  |
| 已处置            | 处置流程审批通过  |

### 4.6 资产来源枚举

| 枚举值   | 说明   |
| ----- | ---- |
| 外购    | 外部采购 |
| 租赁    | 租赁使用 |
| 融资租赁  | 融资租赁 |
| 借用    | 借用   |
| 自研    | 自主研发 |

### 4.7 导出规则

1. 选中导出：仅导出当前页选中的数据，不支持跨页导出
2. 全量导出：导出当前筛选条件下的所有数据
3. 未选择数据时弹出确认提示

### 4.8 BOM清单数据同步规则

1. MES系统出厂BOM数据在车辆生产完毕后一次性同步，同步后数据锁定不再变更
2. 售后系统零部件变更数据在每次维修完成后增量推送，持续追加
3. 当前BOM视图数据为计算结果：以出厂BOM为基础 + 售后变更按件号覆盖
4. BOM清单Tab仅当车辆"资产来源"为"自研"时显示
5. BOM数据同步失败时，系统标记同步异常状态，支持管理员手动触发重新同步
6. 售后系统推送的BOM变更数据自动同步记录至变更日志Tab

```
    │   ├── 调度信息 → 查看调度记录列表 → 查询/导出
    │   ├── BOM清单 → 仅自研车辆可见，查看当前BOM/变更记录
    │   ├── 变更日志 → 查看操作变更记录
    │   └── 维保/合规/风险/费用 → 置灰，暂未开放
    ├── 点击编辑 → 当前Tab进入编辑态 → 修改 → 确定/取消
    └── 点击关闭 → 返回列表页
```

### 5.3 车辆分配流程

```
用户勾选车辆 → 点击车辆分配
  → 校验：
  ├── 未勾选 → 提示"未选择车辆"
  ├── 存在非"待分配"车辆 → 提示"已分配的车辆请走车辆调拨流程"
  └── 校验通过 → 打开分配弹窗
    → 选择分配至项目（全选/单选）
    → 点击确定 → 执行分配 → 关闭弹窗 → 列表刷新
    → 点击取消 → 关闭弹窗
```

### 5.4 车辆状态变更流程

```
用户勾选车辆 → 点击状态变更
  → 校验：
  ├── 未勾选 → 提示"未选择车辆"
  └── 校验通过 → 打开状态变更弹窗
    → 选择变更后状态（全选/单选）
    → 点击确定 → 校验所有车辆均已选择状态
      ├── 存在未选择 → 提示"请为所有车辆选择变更后状态"
      └── 校验通过 → 执行状态变更 → 记录变更日志 → 关闭弹窗 → 列表刷新
    → 点击取消 → 关闭弹窗
```
