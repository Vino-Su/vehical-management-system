## 1. 业务背景与使用角色

### 1.1 业务背景

调度管理模块是车辆管理系统中负责车辆跨项目流转的核心业务模块。当某项目存在闲置可调出的车辆，而其他项目需要车辆支持时，通过本模块实现车辆从移出方到接收方的完整流转闭环管理。模块涵盖**调度车辆库**（可用车辆查询与发起调拨）和**调度记录管理**（调拨全流程跟踪与审批）两大子模块，支撑车辆调拨从申请、审批、出库到单车交接确认的全生命周期管理。

本模块与关联模块的关系：
- **车辆建档模块**：调度车辆库的数据来源，仅展示车辆状态为"在库-闲置可调出"的车辆
- **车辆资产管理模块**：调度车辆库点击"详情"跳转至车辆资产详情页
- **项目管理模块**：接收项目、流转前后项目的数据来源

### 1.2 使用角色

| 角色       | 调度模块中的职责                          | 客户端 |
| ---------- | ---------------------------------------- | ------ |
| 管理员      | 查询可用车辆库、发起调度申请、查看调度记录、审批操作、导出 | PC     |
| 项目经理    | 查询可用车辆库、发起调度申请、查看调度记录、审批操作、导出 | PC     |
| 财务人员    | 查看调度记录、导出数据                     | PC     |
| 移出方人员  | 对自己项目的车辆执行出库审批、出库信息完善   | PC     |
| 接收方人员  | 对流转至自己项目的车辆执行交接审批、单车交接信息确认、结束任务 | PC     |
| 管理层      | 查看调度记录                              | PC     |

### 1.3 核心目标与价值

| 目标维度   | 说明                                       |
| ---------- | ------------------------------------------ |
| 资源盘活   | 汇聚全公司范围内闲置可调出车辆，提升车辆利用率 |
| 流程闭环   | 覆盖申请-审批-出库-单车交接完整流程，确保权责清晰 |
| 状态可视   | 调度记录统一汇聚，各节点状态一目了然          |
| 数据溯源   | 记录审批流程日志，支持全流程追溯              |
| 权限管控   | 按角色和项目维度控制操作权限，确保数据安全     |
| 单车验收   | 支持逐辆车进行交接确认，确保车辆交接质量       |

## 3. 功能详细设计

### 3.1 F01 调度车辆库列表页

#### 3.1.1 页面定位

调度车辆库是调度管理的入口页面，汇聚公司范围内车辆状态为**"在库-闲置可调出"**的所有车辆，支持用户查询可用车辆并批量发起调拨申请。

#### 3.1.2 页面结构

页面自上而下分为三个区域：
1. **查询筛选区**：提供查询条件输入与操作按钮
2. **数据列表区**：以表格形式展示可用车辆数据
3. **分页区**：展示分页控件与数据统计

#### 3.1.3 查询筛选区

查询条件与查询按钮放置在同一行显示。

| 元素           | 类型       | 配置说明                                                         |
| -------------- | ---------- | ---------------------------------------------------------------- |
| 车架号         | 文本输入框 | 默认显示，支持模糊搜索                                           |
| 车型名称（中） | 文本输入框 | 默认显示，支持模糊搜索                                           |
| 车型版本       | 文本输入框 | 默认显示，支持模糊搜索                                           |
| 网联编号       | 文本输入框 | 默认显示，支持模糊搜索                                           |
| 车型编码       | 文本输入框 | 收起，展开后显示，支持模糊搜索                                   |
| 车牌号         | 文本输入框 | 收起，展开后显示，支持模糊搜索                                   |
| 车辆型号       | 文本输入框 | 收起，展开后显示，支持模糊搜索                                   |
| 资产来源       | 下拉单选   | 收起，展开后显示，枚举值：外购、租赁、融资租赁、借用、自研       |
| 归属项目       | 下拉单选   | 收起，展开后显示，选项来自项目列表                               |
| 运营负责人     | 文本输入框 | 收起，展开后显示，支持模糊搜索                                   |

| 新建调度申请按钮 | 主按钮   | 点击打开新建调度申请弹窗；需先勾选车辆；仅管理员、项目经理可见   |
| 导出按钮       | 次按钮     | 点击执行导出；仅管理员、项目经理、财务人员可见                   |

**查询条件显示规则：**
- 默认显示前4个查询条件：车架号、车型名称（中）、车型版本、网联编号
- 其余查询条件收起，点击"展开"后显示全部查询条件

- 表头存在全选复选框，勾选后选中当前页所有行
- 支持单行勾选
- 勾选状态用于新建调度申请和导出操作

#### 3.1.5 分页区

| 配置项     | 规则                                     |
| ---------- | ---------------------------------------- |
| 每页条数选项 | 10、20、50、100，默认10                  |

| 翻页控件   | 首页、上一页、页码、下一页、末页         |
| 切换每页条数 | 下拉选择，切换后从第一页重新加载         |

#### 3.1.6 新建调度申请交互

**触发方式：** 用户勾选车辆后，点击"新建调度申请"按钮。

**前置校验：**

| 校验条件                 | 提示信息               | 阻断操作 |
| ------------------------ | ---------------------- | -------- |
| 用户未勾选任何车辆       | "当前未选择任何车辆"   | 是       |

**校验通过后：** 打开新建调度申请弹窗，车辆清单中自动带入已勾选的车辆信息。

#### 3.1.7 导出功能

**触发方式：** 用户点击列表页"导出"按钮。

**导出规则：**

| 触发条件             | 行为                                                         |
| -------------------- | ------------------------------------------------------------ |
| 用户选中了特定数据   | 导出用户选择的数据，不支持跨页导出                           |
| 用户未选择特定数据   | 弹出提示："是否导出所有数据？"，用户选择"是"则导出当前筛选条件下的所有数据，选择"否"则不执行操作 |

**导出权限：** 管理员、项目经理、财务人员。

**导出格式：** .xlsx，文件命名规则：调度车辆库_YYYYMMDD.xlsx。

### 3.2 F02 新建调度申请

#### 3.2.1 触发方式

**方式一：** 用户在调度车辆库列表页勾选车辆后，点击"新建调度申请"按钮。
**方式二：** 用户在调度记录列表页点击"新建调度申请"按钮（无需预勾选车辆）。

#### 3.2.2 弹窗规格

| 属性   | 值                    |
| ------ | --------------------- |
| 宽度   | 900px                 |
| 最大宽度 | 90vw                  |
| 位置   | 居中弹出              |
| 遮罩   | 半透明，点击遮罩关闭弹窗 |
| 标题   | 新建调度申请          |

#### 3.2.3 表单字段

弹窗内表单分为两部分：**申请信息区**和**车辆清单区**。

**申请信息区**

| 字段名       | 字段类型   | 是否必填 | 校验规则                     | 默认值 | 说明                                                         |

| 车辆编号     | 文本       | —        | —                            | —      | 只读展示，从车辆清单自动汇总                                 |
| 车架号       | 文本       | —        | —                            | —      | 只读展示，从车辆清单自动汇总                                 |
| 当前项目     | 文本       | —        | —                            | —      | 只读展示，车辆清单中车辆的归属项目（需保持一致）             |
| 接收项目     | 下拉单选   | 是       | —                            | —      | 从当前用户权限范围内的项目列表获取                           |
| 接收人       | 下拉单选   | 是       | —                            | —      | 指接收项目内的人员名称，支持搜索                             |
| 预计接收时间 | 日期选择   | 是       | 必须大于等于当前日期           | —      | 精确到日                                                     |
| 周转用途     | 下拉单选   | 是       | —                            | —      | 枚举值：项目流转                                             |
| 审批人       | 下拉单选   | 是       | —                            | —      | 从当前项目内的人员名称中选择，支持搜索                       |

**车辆清单区**

车辆清单以表格形式展示已选车辆的基础信息。

| 序号 | 字段名       | 说明                   |
| ---- | ------------ | ---------------------- |
| 1    | 车辆编号     |                        |
| 2    | 车架号       |                        |
| 3    | 车型名称（中）|                        |
| 4    | 归属项目     |                        |
| 5    | 操作         | 删除（移出该车辆）     |

**车辆清单交互：**
- 点击"删除"：从车辆清单中移出该车辆
- 点击"添加"：打开车辆选择弹窗，支持搜索、勾选车辆，添加至车辆清单中
- 车辆清单至少保留一条记录（最后一条不可删除）

#### 3.2.4 交互流程

1. 用户触发新建调度申请 → 打开弹窗
   - 方式一：车辆清单自动带入已勾选的车辆信息
   - 方式二：车辆清单为空，需用户点击"添加"选择车辆
2. 用户填写申请信息区字段，必填项标红色星号
3. 用户可在车辆清单区添加或删除车辆
4. 点击"提交"：
   - 系统执行表单校验（必填项、业务规则校验）
   - 校验成功 → 显示"提交成功"Toast提示，关闭弹窗，调度记录列表刷新
   - 校验失败 → 在对应字段下方或弹窗顶部显示错误提示，弹窗不关闭
5. 点击"取消" → 关闭弹窗，不保留申请信息
6. 点击"关闭"（右上角×） → 关闭弹窗，不保留申请信息

### 3.4 F04 调度申请详情页

#### 3.4.1 页面定位

展示某一调度申请的完整流转信息，以步骤条形式呈现调度申请、车辆出库、车辆交接三个环节，右侧展示审批流程日志，实现申请全流程的可视化追踪。

#### 3.4.2 页面结构

页面分为左右两栏布局：
1. **左侧详情区**（占70%宽度）：以步骤条形式展示三个环节的详细信息
2. **右侧流程日志区**（占30%宽度）：展示当前申请任务的审批流程日志

#### 3.4.3 操作栏

**顶部操作栏：**

| 按钮   | 类型   | 行为说明               |
| ------ | ------ | ---------------------- |
| 关闭   | 次按钮 | 关闭当前详情页         |

**底部固定操作栏：**

底部操作栏**始终由任务状态决定**，与当前选中步骤解耦。无论用户查看哪个步骤的内容，底部始终显示当前任务阶段对应的操作按钮，便于审批人在查看上下文信息的同时直接执行审批操作。

| 任务状态       | 底部操作栏按钮                              | 说明                                                         |
| -------------- | ------------------------------------------ | ------------------------------------------------------------ |
| 已提交         | 同意出库（主按钮）、不同意出库（次按钮）     | 移出方在详情页直接完成出库审批；默认显示调度申请步骤内容       |
| 已撤回         | 再次提交（主按钮）、取消（次按钮）           | 进入编辑态后可修改并再次提交；默认显示调度申请步骤内容         |
| 出库审批退回   | 再次提交（主按钮）、取消（次按钮）           | 进入编辑态后可修改并再次提交；默认显示调度申请步骤内容         |
| 待出库         | 确认出库（主按钮）                           | 填写出库信息后可确认出库；默认显示车辆出库步骤内容             |
| 已出库         | 同意交接（主按钮）、不同意交接（次按钮）     | 接收方在详情页直接完成交接审批；默认显示车辆出库步骤内容       |
| 待交接         | 任务结束（主按钮）                           | 所有车辆交接确认完成后可结束任务；默认显示车辆交接步骤内容     |
| 已完成         | 关闭（次按钮）                               | 无特殊操作                                                   |
| 已关闭         | 关闭（次按钮）                               | 无特殊操作                                                   |

**底部操作栏交互规则：**
1. 固定在页面底部，始终可见
2. 按钮右侧对齐
3. 用户切换选中步骤后，底部操作栏**不随步骤切换而变化**，始终由任务状态决定
4. 点击"同意出库"/"不同意出库"/"同意交接"/"不同意交接"后，与列表页弹窗审批效果一致（状态变更、记录日志）

#### 3.4.4 步骤条设计

步骤条分为三个环节，依次为：
1. **调度申请**（第一步）
2. **车辆出库**（第二步）
3. **车辆交接**（第三步）

**步骤状态规则：**

| 步骤       | 当前任务状态范围                               | 步骤状态   |
| ---------- | ---------------------------------------------- | ---------- |
| 调度申请   | 所有状态                                       | 已完成（绿色） |
| 车辆出库   | 已提交、已撤回、出库审批退回                     | 待处理（灰色，不可点击） |
| 车辆出库   | 待出库、已出库、待交接、已完成                   | 已完成（绿色，可点击） |
| 车辆出库   | 已关闭                                         | 已完成（绿色，可点击） |
| 车辆交接   | 已完成                                         | 已完成（绿色，可点击） |
| 车辆交接   | 待交接                                         | 进行中（蓝色） |
| 车辆交接   | 已关闭                                         | 已关闭（灰色，可点击） |
| 车辆交接   | 其他状态                                       | 待处理（灰色，不可点击） |

**步骤选中样式规则：**

步骤条需要同时表达两种状态：
1. **流程状态**：该步骤在整体流程中的完成状态（已完成/进行中/待处理/已关闭）
2. **选中状态**：用户当前正在查看该步骤的详情内容

选中态使用**蓝色实线边框**包裹整个步骤项（step-item），与流程状态样式叠加显示：
- 已完成的步骤 + 选中 → 绿色圆圈 + 蓝色边框
- 进行中的步骤 + 选中 → 蓝色圆圈 + 蓝色边框（默认选中当前进行中的步骤）
- 已关闭的步骤 + 选中 → 灰色圆圈 + 蓝色边框

**步骤点击规则：**
- 已完成的步骤可点击，点击后切换至对应步骤的详情内容，该步骤获得选中态
- 待处理的步骤置灰显示，不可点击，无选中态
- 进行中的步骤高亮显示，可点击，默认处于选中态
- 用户切换选中步骤后，底部操作栏根据当前选中步骤和任务状态动态更新操作按钮

#### 3.4.5 右侧流程日志区

**记录信息字段：**

| 字段名   | 说明                      |
| -------- | ------------------------- |
| 节点名称 | 如：调度申请提交、出库审批、交接审批 |
| 审批人   | 执行该节点操作的人员名称   |
| 审批时间 | 格式：YYYY-MM-DD HH:mm:ss |
| 审批结果 | 如：提交成功、同意、不同意、退回等 |

**显示规则：**
- 按审批时间由早到晚顺序排列（时间线从上到下）
- 最新节点高亮显示
- 支持滚动查看全部日志

### 3.5 F05 车辆出库

#### 3.5.2 出库信息完善（单车级）

**触发方式：** 用户在调度记录列表页，对待出库状态的记录点击"出库信息完善"按钮；或在调度申请详情页点击"车辆出库"步骤按钮。

1. **上方整批信息区**：展示整批车辆的公共出库信息（发车时间、发车方式）
2. **中间车辆出库区**：以可折叠行表格展示本次调度涉及的所有车辆，支持逐辆车进行出库信息确认
3. **底部操作栏**：确认出库按钮

**整批出库信息（公共字段）：**

| 字段名   | 字段类型 | 是否必填 | 说明             |

**车辆列表字段：**

| 序号 | 字段名       | 字段标识        | 说明                       |

| 3    | 车型名称（中）| model_name     | 只读展示                   |

| 字段名         | 字段类型   | 是否必填 | 校验规则                  | 说明                       |
| -------------- | ---------- | -------- | ------------------------- | -------------------------- |
| 移出前车辆照片 | 图片上传   | 是       | 支持jpg/png，单文件≤5MB   | 每辆车独立上传             |
| 车辆出库单     | 附件上传   | 是       | 支持常见文档格式           | 每辆车独立上传             |
| 说明           | 文本域     | 否       | 最大500字                 | 该车的出库备注说明         |

5. 用户点击某辆车的"出库确认"按钮，该行下方展开出库表单

   - 表单收起，列表刷新
   - 显示"保存成功"Toast提示

- 鼠标悬停置灰的"确认出库"按钮时，Tooltip提示具体原因
- 点击"确认出库"后：
  - 任务状态变更为"已出库"
  - 记录审批日志（节点名称：确认出库，审批结果：出库成功）
  - 显示"出库成功"Toast提示

点击某辆车的"交接确认"按钮后，该行下方展开交接表单。

| 字段名         | 字段类型   | 是否必填 | 校验规则                  | 说明                       |
| -------------- | ---------- | -------- | ------------------------- | -------------------------- |
| 接收时车辆照片 | 图片上传   | 是       | 支持jpg/png，单文件≤5MB   | 每辆车独立上传             |
| 说明           | 文本域     | 否       | 最大500字                 | 该车的交接备注说明         |
| 是否确认接收   | 单选       | 是       | 选项：是/否               | 选择"否"时该车标记为拒绝接收 |

**单车交接交互流程：**

1. 用户进入车辆交接步骤详情页
2. 页面顶部显示步骤条，"车辆交接"步骤高亮
3. 车辆列表展示所有待交接车辆，初始状态均为"未交接"
4. 用户点击某辆车的"交接确认"按钮，该行下方展开交接表单
5. 用户填写接收时车辆照片、说明、是否确认接收
6. 点击"保存"：
   - 系统执行表单校验
   - 校验通过 → 该车交接状态更新：
     - "是否确认接收"选择"是" → 状态变为"已交接"
     - "是否确认接收"选择"否" → 状态变为"拒绝接收"
   - 表单收起，列表刷新
   - 显示"保存成功"Toast提示
7. 点击"取消" → 表单收起，不保存数据

**任务结束规则：**

- 页面底部显示"任务结束"按钮
- "任务结束"按钮可用条件：**所有车辆均已完成交接确认**（交接状态为"已交接"或"拒绝接收"）
- 若存在"未交接"状态的车辆，"任务结束"按钮置灰不可用
- 鼠标悬停置灰的"任务结束"按钮时，Tooltip提示："请完成所有车辆的交接确认"

**点击"任务结束"：**

1. 系统校验所有车辆交接状态均不为"未交接"
2. 校验通过 → 任务状态变更为"已完成"
3. 记录审批日志（节点名称：交接完成，审批结果：任务结束）
4. 显示"任务已结束"Toast提示，关闭页面

**点击"关闭"：** 关闭当前页面。

**点击其他步骤按钮（如已完成）→** 跳转至相应页面。

**拒绝接收车辆的处理说明：**

若有车辆"是否确认接收"选择"否"（拒绝接收），该车状态标记为"拒绝接收"。拒绝接收不影响整个调度任务的完结，任务仍可正常结束。拒绝接收的车辆由接收方与移出方线下人工协调处理，系统仅记录拒绝状态，不参与后续流程流转。

## 4. 业务规则汇总

### 4.1 任务状态枚举

| 枚举值         | 说明                               | 可执行操作                     |
| -------------- | ---------------------------------- | ------------------------------ |
| 已提交         | 调度申请已提交，等待移出方出库审批   | 撤回、出库审批、详情            |
| 已撤回         | 申请人主动撤回已提交的申请           | 再次提交、详情                  |
| 出库审批退回   | 移出方不同意出库，申请被退回         | 再次提交、详情                  |
| 待出库         | 移出方同意出库，等待填写出库信息     | 出库信息完善、详情              |
| 已出库         | 出库信息已完善，等待接收方交接审批   | 交接审批、详情                  |
| 待交接         | 接收方同意接收，等待逐辆车进行交接确认 | 交接信息完善、详情              |
| 已完成         | 所有车辆交接确认完成，调度任务结束   | 详情                           |
| 已关闭         | 交接审批不同意，流程直接结束         | 详情                           |

**状态流转图：**

```
已提交 ──同意──→ 待出库 ──确认出库──→ 已出库 ──同意──→ 待交接 ──全部确认──→ 已完成
   │                │                                          │
   │                │                                          │
  撤回            不同意                                      不同意
   │                │                                          │
   ▼                ▼                                          ▼
已撤回 ──再次提交──┘                                已关闭（流程结束）
   │
出库审批退回 ──再次提交──┘
```

### 4.2 调度申请提交校验规则

| 校验项           | 校验规则                                                     | 失败提示                                | 阻断操作 |
| ---------------- | ------------------------------------------------------------ | --------------------------------------- | -------- |
| 车辆状态校验     | 所选车辆状态必须均为"在库-闲置可调出"                        | 弹窗告知校验失败的车辆信息               | 是       |
| 项目一致性校验   | 同一申请中不允许选择属于不同项目的车辆                        | 弹窗告知"所选车辆归属项目不一致"       | 是       |
| 必填项校验       | 用户角色、接收项目、接收人、预计接收时间、周转用途、审批人不可为空 | "XX不能为空"                           | 是       |
| 车辆清单非空校验 | 车辆清单中至少保留一条车辆记录                                | "车辆清单不能为空"                     | 是       |
| 预计接收时间校验 | 预计接收时间必须大于等于当前日期                              | "预计接收时间不能早于今天"             | 是       |
| 接收项目权限校验 | 接收项目必须在当前用户权限范围内                              | "无权选择该项目作为接收项目"           | 是       |

### 4.3 调度申请编辑规则（再次提交）

1. 仅"已撤回"和"出库审批退回"状态的记录支持"再次提交"
2. 点击"再次提交"后，进入调度申请编辑态，页面同新建调度申请弹窗
3. 编辑态下可修改所有申请信息字段和车辆清单
4. 点击"提交"：重新执行提交校验，校验通过后状态变更为"已提交"
5. 点击"取消"：退出编辑态，返回详情页，不保存修改

### 4.4 撤回规则

1. 仅"已提交"状态的记录支持撤回
2. 撤回条件：下一环节（出库审批）尚未处理
3. 若下一环节已处理（已审批），则"撤回"按钮不可见或置灰
4. 撤回后任务状态变更为"已撤回"
5. 记录审批日志（节点名称：撤回申请，审批结果：已撤回）

### 4.5 审批权限规则

| 操作         | 权限范围说明                     |
| ------------ | -------------------------------- |
| 出库审批     | 移出方项目的人员或管理员可执行   |
| 出库信息完善 | 移出方项目的人员或管理员可执行   |
| 交接审批     | 接收方项目的人员或管理员可执行   |
| 交接信息确认 | 接收方项目的人员或管理员可执行   |
| 结束任务     | 接收方项目的人员或管理员可执行   |

### 4.6 流程日志记录规则

1. 每个关键节点操作均需记录流程日志
2. 日志记录字段：节点名称、审批人、审批时间、审批结果
3. 节点覆盖：调度申请提交、撤回、再次提交、出库审批、确认出库、交接审批、单车交接确认、任务结束
4. 日志按时间顺序展示，不可篡改
