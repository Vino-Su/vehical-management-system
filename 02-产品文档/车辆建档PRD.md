## 1. 业务背景与使用角色

### 1.1 业务背景

车辆管理系统是一个从0到1构建的企业级平台，围绕车辆全生命周期管理，对公司所有车辆（外购、租赁、融资租赁、借用、自研等）进行统一数字化管理。车辆建档模块是系统的数据入口，承担车辆资产信息的录入与记录职责，是后续资产管理、调拨、运行、退出等全流程的数据基础。

### 1.2 使用角色

| 角色       | 建档模块中的职责                     | 客户端   |
| -------- | ----------------------------- | ----- |
| 管理员      | 单个录入、批量导入、删除所有车辆、查看详情        | PC+APP |
| 项目经理     | 单个录入、批量导入、删除自己导入的车辆、查看详情     | PC+APP |
| 财务人员     | 维护资产分类、合同主体、固定资产编码、资产属性等财务字段 | PC    |
| 管理层      | 查看车辆建档信息                     | PC    |
| 车队队长/司机  | 查看车辆建档信息（数据范围限自身项目）          | APP   |

### 1.3 核心目标与价值

| 目标维度   | 说明                                |
| ------ | --------------------------------- |
| 数据入口   | 提供手动录入（单个/批量）和接口接入两种数据来源通道        |
| 数据完整性  | 覆盖车辆基础信息、归属信息、车型信息、行驶证、硬件/软件/摄像头等全维度字段 |
| 数据可溯源  | 记录建档人、建档时间、建档方式，支持后续溯源查询         |
| 数据准确性  | 通过VIN码唯一性校验、字段合法性校验、权限校验确保数据质量    |

## 3. 功能详细设计

### 3.1 F01 建档列表页

#### 3.1.1 页面定位

统一管理车辆建档数据的核心页面，是用户进入车辆建档模块的默认页面。

#### 3.1.2 页面结构

页面自上而下分为三个区域：
1. **查询筛选区**：提供查询条件输入与操作按钮
2. **数据列表区**：以表格形式展示车辆建档数据
3. **分页区**：展示分页控件与数据统计

#### 3.1.3 查询筛选区

| 元素     | 类型         | 配置说明                                              |
| ------ | ---------- | ------------------------------------------------- |
| 建档时间   | 日期范围选择器    | 选择起止日期，默认近半年                                      |
| 建档人    | 文本输入框      | 支持模糊搜索                                            |
| 建档方式   | 下拉单选       | 枚举值：全部、单个录入、批量导入、采购系统、MES系统；默认"全部"                |
| 查询按钮   | 主按钮        | 点击执行查询，列表刷新为查询结果                                  |
| 重置按钮   | 次按钮        | 点击重置查询条件至默认状态（建档时间=近半年，建档方式=全部，建档人=空）             |
| 单个录入按钮 | 主按钮        | 点击打开单个录入弹窗；仅管理员、项目经理可见                            |
| 批量导入按钮 | 次按钮        | 点击打开批量导入弹窗；仅管理员、项目经理可见                            |

#### 3.1.4 数据列表区

**列表显示字段（从左到右）：**

| 序号 | 字段名      | 字段标识         | 宽度建议   | 对齐方式 | 说明       |
| -- | -------- | ------------ | ------ | ---- | -------- |
| 1  | 车架号      | vin_code     | 200px  | 左对齐  | VIN码     |
| 2  | 出厂编号     | factory_no   | 160px  | 左对齐  |          |
| 3  | 网联编号     | iot_code     | 160px  | 左对齐  |          |
| 4  | 车型名称（中）  | model_name   | 180px  | 左对齐  |          |
| 5  | 归属项目     | project_name | 160px  | 左对齐  |          |
| 6  | 建档人      | creator      | 120px  | 左对齐  |          |
| 7  | 建档时间     | create_time  | 180px  | 左对齐  | 格式：YYYY-MM-DD HH:mm:ss |
| 8  | 建档方式     | create_type  | 120px  | 左对齐  | 枚举值展示    |
| 9  | 操作       | actions      | 120px  | 居中   | 详情、删除    |

**排序规则：** 默认按建档时间由近到远倒序排列。

**操作列按钮：**

| 操作   | 触发行为          | 可见角色           |
| ---- | ------------- | -------------- |
| 详情   | 跳转至车辆档案详情页    | 所有角色           |
| 删除   | 弹出二次确认弹窗后逻辑删除 | 管理员（所有）、项目经理（仅自己导入的） |

**行交互：** 行hover高亮。

#### 3.1.5 分页区

| 配置项     | 规则                                |
| ------- | --------------------------------- |
| 每页条数选项  | 10、20、50、100，默认10                |
| 显示信息    | 总记录数、当前页码/总页数                    |
| 翻页控件    | 首页、上一页、页码、下一页、末页                 |
| 切换每页条数  | 下拉选择，切换后从第一页重新加载                 |

#### 3.1.6 导出功能（新增）

**触发方式：** 用户点击列表页"导出"按钮。

**导出规则：**

| 触发条件         | 行为                                      |
| ------------ | --------------------------------------- |
| 用户选中了特定数据    | 导出用户选择的数据，不支持跨页导出                       |
| 用户未选择特定数据    | 弹出提示："是否导出所有数据？"，用户选择"是"则导出当前筛选条件下的所有数据，选择"否"则不执行操作 |

**列表复选框交互：**
- 表头存在全选复选框，勾选后选中当前页所有行
- 支持单行勾选
- 勾选状态用于导出操作

**导出权限：** 管理员、项目经理、财务人员。

**导出格式：** .xlsx，文件命名规则：车辆建档_YYYYMMDD.xlsx

#### 3.2.4 交互流程

1. 用户点击"单个录入"按钮 → 打开录入弹窗
2. 弹窗内按分组展示表单字段，必填项标红色星号
3. 用户选择"车型名称（中）"后，"车型版本"下拉选项自动填充该车型的可选版本，其余车型信息字段暂不填充
4. 用户选择"车型版本"后，车型信息分组中的关联字段自动填充（只读）
4. 用户选择"是否支持联网"为"是"时，"网联编号"字段显示并变为必填
5. 用户填写"注册日期"后，"车龄"字段自动计算显示
6. 硬件信息、软件信息、摄像头信息分组支持"添加行"和"删除行"操作
7. 点击"确认"：
   - 系统执行表单校验（必填项、格式校验、VIN码唯一性校验）
   - 校验成功 → 显示"录入成功"Toast提示，关闭弹窗，列表刷新
   - 校验失败 → 在对应字段下方显示红色错误提示，弹窗不关闭
8. 点击"取消" → 关闭弹窗，不保存任何数据

| 已选文件信息   | 文本显示   | 显示已选文件名、大小                            |

#### 3.3.4 交互流程

2. 用户可先点击"模板下载"下载导入模板
3. 用户通过点击或拖拽方式上传Excel文件
4. 上传后显示文件名和大小，支持移除已选文件重新选择
5. 点击"确认"：
   - 系统执行批量校验（详见3.7导入校验规则）
   - 校验全部成功 → 显示"导入成功"Toast提示，关闭弹窗，列表刷新
   - 校验存在失败 → 弹出校验失败弹窗（详见3.8校验失败提示），批量数据均不导入

### 3.5 F05 删除

#### 3.5.1 触发方式

### 3.8 F08 校验失败提示

- 弹窗不关闭，用户可修正后重新提交

  - 底部提示："请修正后重新导入"
  - 关闭按钮

## 4. 业务规则汇总

### 4.1 数据来源

| 来源方式   | 适用场景          | 说明                    |
| ------ | ------------- | --------------------- |
| 单个录入   | 少量车辆数据录入      | 通过弹窗表单逐条录入            |
| 批量导入   | 大量车辆数据批量录入    | 通过Excel模板导入，上限5000行   |
| 采购系统接入 | 外采车辆数据        | 通过接口从采购系统同步           |
| MES系统接入 | 工厂自研车辆数据      | 通过接口从MES系统同步          |

### 4.2 建档方式枚举

| 枚举值   | 说明       |
| ----- | -------- |
| 单个录入  | 用户手动逐条录入 |
| 批量导入  | 用户通过模板导入 |
| 采购系统  | 接口自动同步   |
| MES系统 | 接口自动同步   |

### 4.3 车辆状态枚举

| 枚举值            | 说明          |
| -------------- | ----------- |
| 正常在用           | 车辆正常使用中     |
| 闲置-验标留用/备用     | 闲置但保留使用     |
| 闲置-可调出         | 闲置可调配至其他项目  |
| 闲置-维修(时间小于2周)  | 短期维修中       |
| 维修中(时间大于2周)    | 长期维修中       |
| 报废流程中          | 正在走报废审批     |
| 库存             | 车辆在库未分配     |
| 待分配            | 批量导入时归属项目为空 |

### 4.4 资产来源枚举

| 枚举值   | 说明   |
| ----- | ---- |
| 外购    | 外部采购 |
| 租赁    | 租赁使用 |
| 融资租赁  | 融资租赁 |
| 借用    | 借用   |
| 自研    | 自主研发 |

### 4.5 资产属性枚举

| 枚举值      | 说明      |
| -------- | ------- |
| 非机动车     | —       |
| 运营车辆-电动  | 电动运营车辆  |
| 运营车辆-油动  | 燃油运营车辆  |
| 运营车辆-气动  | 气动运营车辆  |
| 智能驾驶设备   | 自动驾驶车辆  |
| 运输设备     | 运输类车辆   |
| 乘用车      | 乘用类车辆   |

### 4.6 删除规则

1. 仅支持用户删除由自己导入的车辆建档信息（管理员除外）
2. 管理员可删除所有车辆建档信息
3. 仅做逻辑删除，不物理删除数据
4. 删除前需二次确认

### 4.7 联动规则

| 触发字段   | 触发条件     | 联动行为              |
| ------ | -------- | ----------------- |
| 是否支持联网 | 选择"是"    | 网联编号字段显示并变为必填     |
| 是否支持联网 | 选择"否"    | 网联编号字段隐藏并清空       |

## 8. 关联说明

本模块（车辆建档）是车辆管理系统的数据入口模块，与以下模块存在关联：

| 关联模块   | 关联关系                  | 说明                    |
| ------ | --------------------- | --------------------- |
| 车型管理   | 车型信息字段引用车型表数据         | 车型名称选择后自动获取关联字段       |
| 项目管理   | 归属项目字段引用项目列表数据        | 项目数据来源待确认             |
| 车辆档案   | 建档数据是档案数据的基础          | 详情查看跳转至档案详情页          |
| 采购系统   | 外采车辆数据通过接口同步          | 接口字段映射待联调确认           |
| MES系统  | 自研车辆数据通过接口同步          | 接口字段映射待联调确认           |
