## 1. 业务背景与使用角色

### 1.1 业务背景

车辆在购买时通常附带售后政策、质保期限、售后服务商等信息，这些信息对于后续的质保判断、维修保养决策、费用核算等至关重要。当前车辆资产详情页中缺少售后维保相关信息的承载，导致用户无法快速了解车辆的质保状态、售后服务渠道和维保计划。

售后维保信息模块作为车辆资产详情页的新增Tab页，承接车辆售后权益信息和维保记录的展示与管理，是车辆全生命周期管理中"资产管理→运行→退出"环节的重要补充。

### 1.2 使用角色

| 角色       | 售后维保模块中的职责                     | 客户端   |
| -------- | ----------------------------- | ----- |
| 管理员      | 查看、编辑所有车辆的售后维保信息              | PC+APP |
| 项目经理     | 查看、编辑权限范围内车辆的售后维保信息          | PC+APP |
| 财务人员     | 查看售后维保信息（质保状态影响资产价值判断）       | PC    |
| 管理层      | 查看售后维保信息                       | PC    |
| 车队队长/司机  | 查看自身项目车辆的售后维保信息               | APP   |

### 1.3 核心目标与价值

| 目标维度    | 说明                              |
| ------- | ------------------------------- |
| 质保可视化   | 一眼看清车辆质保状态，判断是否在质保期内           |
| 服务可触达   | 明确售后服务商和联系方式，车辆故障时可快速联系        |
| 维保可计划   | 基于维保计划自动计算下次保养提醒，避免逾期保养        |
| 数据可溯源   | 售后信息编辑自动记录变更日志，保证数据可审计          |
| 后续可扩展   | 预留维修记录和保养记录的数据结构，对接售后系统后可自动填充  |

## 3. 功能详细设计

### 3.1 F01 售后维保Tab页

#### 3.1.1 页面定位

在车辆资产详情页中新增"售后维保"Tab，承载车辆的售后权益信息和维保记录数据。采用子Tab标签形式组织，分为"售后信息"、"维修信息"、"保养信息"三个子Tab。

#### 3.1.2 Tab位置

位于BOM清单Tab之后、原"维保信息"Tab位置。新增"售后维保"Tab后，原"维保信息"Tab移除（由售后维保Tab替代）。

**调整后Tab顺序：**

| 序号 | Tab名称  | 本期建设 | 显示条件             |
| -- | ------ | ---- | ---------------- |
| 1  | 基础信息   | 是    | 始终显示             |
| 2  | 调度信息   | 是    | 始终显示             |
| 3  | BOM清单  | 是    | 仅当车辆"资产来源"为"自研"时显示 |
| 4  | **售后维保** | **是** | **始终显示**           |
| 5  | 合规信息   | 否    | 始终显示（置灰）         |
| 6  | 风险信息   | 否    | 始终显示（置灰）         |
| 7  | 费用信息   | 否    | 始终显示（置灰）         |
| 8  | 变更日志   | 是    | 始终显示             |

#### 3.1.3 子Tab定义

售后维保Tab内部包含三个子Tab标签：

| 序号 | 子Tab名称 | 本期建设 | 说明                                       |
| -- | ------- | ---- | ---------------------------------------- |
| 1  | 售后信息    | 是    | 展示质保政策、售后服务商、维保计划、售后附件等静态信息，支持查看和编辑 |
| 2  | 维修信息    | 否    | 展示维修记录列表，本期置灰标注"暂未开放"，对接售后系统后激活       |
| 3  | 保养信息    | 否    | 展示保养记录列表，本期置灰标注"暂未开放"，对接售后系统后激活       |

**默认显示：** 售后信息子Tab。

#### 3.1.4 编辑态规则

1. 售后维保Tab支持编辑，点击详情页顶部"编辑"按钮后，当前子Tab进入编辑态
2. 编辑态下，底部显示固定操作栏，包含"确定"和"取消"按钮
3. 点击"确定"：执行校验，校验成功则保存并记录变更日志，校验失败则显示错误提示
4. 点击"取消"：退出编辑态，不保存任何修改
5. 维修信息和保养信息子Tab为只读（本期不建设，后续对接售后系统后也为只读）
6. 切换到维修信息或保养信息子Tab时，"编辑"按钮隐藏

### 3.2 F05 概览统计条

#### 3.2.1 页面定位

在售后维保Tab顶部展示轻量统计条，帮助用户快速感知车辆质保状态。概览统计条始终可见，不随子Tab切换而隐藏。

#### 3.2.2 显示信息

| 元素       | 类型   | 说明                                       |
| -------- | ---- | ---------------------------------------- |
| 整车质保状态   | 标签   | 使用颜色标签区分：质保期内（绿色）、即将到期（橙色）、已过期（红色）、无质保（灰色）、未录入（灰色虚线）、信息不完整（橙色） |
| 整车质保到期日期 | 日期文本 | 格式：YYYY-MM-DD；未录入时显示"—"                  |
| 剩余/已过期   | 文本   | 质保期内显示"剩余X年X个月"，已过期显示"已过期X年X个月"；未录入时显示"—" |
| 三电质保状态   | 标签   | 仅电动车显示；使用与整车质保状态相同的颜色标签体系              |
| 三电质保到期日期 | 日期文本 | 仅电动车显示；取电池/电机/电控三者中最早到期的日期              |

**示例（质保期内）：**

```
🛡️ 整车质保：[质保期内]  到期 2028-03-15（剩余2年9个月）  │  三电质保：[质保期内]  到期 2033-03-15（剩余7年9个月）
```

**示例（已过期）：**

```
🛡️ 整车质保：[已过期]  到期 2024-03-15（已过期2年2个月）  │  三电质保：[质保期内]  到期 2030-03-15（剩余3年9个月）
```

**示例（未录入）：**

```
🛡️ 整车质保：[未录入]  到期 —  │  三电质保：[未录入]  到期 —
```

#### 3.2.3 三电质保信息显示条件

仅当车辆"动力类别"字段为"纯电动"时，概览统计条中显示三电质保状态和到期日期。非电动车辆仅显示整车质保信息。

### 3.3 F02 售后信息子Tab

#### 3.3.1 页面定位

展示车辆的售后权益静态信息，包括质保政策、售后服务商、维保计划和售后政策附件。查看态为只读展示，编辑态可修改字段。

#### 3.3.2 信息分组

售后信息子Tab围绕车辆的售后权益展开，分为以下5个信息分组：

| 序号 | 分组名称     | 说明                       | 显示条件         |
| -- | -------- | ------------------------ | ------------ |
| 1  | 整车质保信息   | 整体质保政策                   | 始终显示         |
| 2  | 三电质保信息   | 电池/电机/电控质保政策            | 仅动力类别为"纯电动"时显示 |
| 3  | 售后服务商信息  | 服务商联系方式                  | 始终显示         |
| 4  | 维保计划信息   | 保养计划配置                   | 始终显示         |
| 5  | 售后政策附件   | 相关文档附件                   | 始终显示         |

#### 3.3.3 整车质保信息

| 字段名        | 字段标识                   | 字段类型   | 是否必填 | 校验规则                     | 默认值 | 说明                                                         |
| ---------- | ---------------------- | ------ | ---- | ------------------------ | --- | ---------------------------------------------------------- |
| 质保范围       | warranty_scope         | 下拉单选   | 否    | —                        | 根据资产来源自动建议 | 枚举值：整车质保、核心部件质保、无质保；外购→整车质保，自研→核心部件质保，借用→无质保 |
| 质保起始日期     | warranty_start_date    | 日期选择   | 否    | 不能晚于当前日期                 | —   | 质保范围不为"无质保"时可编辑                                           |
| 质保期限（时间）   | warranty_period_year   | 数值输入   | 否    | 正数，精度0.5，单位：年            | —   | 如3、5、8；质保范围不为"无质保"时可编辑                                     |
| 质保期限（里程）   | warranty_period_mileage | 数值输入   | 否    | 正整数，单位：公里                | —   | 如100000；与质保期限（时间）先到为准                                     |
| 质保到期日期     | warranty_end_date      | 日期显示   | —    | 系统计算                     | —   | 质保起始日期 + 质保期限（时间）年数；未填写时间期限时显示"—"                          |
| 当前质保状态     | warranty_status        | 标签显示   | —    | 系统计算                     | 未录入 | 枚举值及计算规则见3.3.4节                                           |
| 质保服务商      | warranty_provider      | 文本输入   | 否    | —                        | 根据资产来源自动建议 | 外购→整车厂商名称，租赁→租赁公司名称，自研→自研公司名称；可与常规售后服务商不同 |

**质保期限说明：**
- 质保期限（时间）和质保期限（里程）为"或"逻辑，先到为准
- 如整车质保"3年或10万公里"，则质保期限（时间）=3，质保期限（里程）=100000
- 两个字段至少填写一个，否则质保状态为"信息不完整"

#### 3.3.4 质保状态计算规则

| 优先级 | 条件                                  | 质保状态   | 标签颜色 |
| --- | ----------------------------------- | ------ | ---- |
| 1   | 质保范围为空                              | 未录入    | 灰色虚线 |
| 2   | 质保范围=无质保                           | 无质保    | 灰色   |
| 3   | 质保范围不为"无质保"，但起始日期或期限均未填           | 信息不完整  | 橙色   |
| 4   | 当前日期 > 质保到期日期（且到期日期不为空）            | 已过期    | 红色   |
| 5   | 当前里程 > 质保期限（里程）（且里程已填写，且当前里程数据可获取） | 已过期    | 红色   |
| 6   | 质保到期日期 - 当前日期 ≤ 30天（且到期日期不为空）      | 即将到期   | 橙色   |
| 7   | 其他                                  | 质保期内   | 绿色   |

**说明：** 当前里程数据需从车辆运行数据中获取，若系统暂无里程数据，则仅按时间条件判断。

#### 3.3.5 三电质保信息

**显示条件：** 仅当车辆"动力类别"为"纯电动"时显示该分组。

| 字段名            | 字段标识                       | 字段类型   | 是否必填 | 校验规则             | 默认值     | 说明                                       |
| -------------- | -------------------------- | ------ | ---- | ---------------- | ------- | ---------------------------------------- |
| 三电质保起始日期       | battery_warranty_start     | 日期选择   | 否    | 不能晚于当前日期         | 质保起始日期  | 未填则默认使用整车质保起始日期                          |
| 电池质保期限（时间）     | battery_warranty_year      | 数值输入   | 否    | 正数，精度0.5，单位：年    | —       | 如8                                       |
| 电池质保期限（里程）     | battery_warranty_mileage   | 数值输入   | 否    | 正整数，单位：公里        | —       | 如150000                                  |
| 电池质保到期日期       | battery_warranty_end       | 日期显示   | —    | 系统计算             | —       | 三电质保起始日期 + 电池质保期限（时间）                    |
| 电机质保期限（时间）     | motor_warranty_year        | 数值输入   | 否    | 正数，精度0.5，单位：年    | —       | 如5                                       |
| 电机质保期限（里程）     | motor_warranty_mileage     | 数值输入   | 否    | 正整数，单位：公里        | —       | 如100000                                  |
| 电机质保到期日期       | motor_warranty_end         | 日期显示   | —    | 系统计算             | —       | 三电质保起始日期 + 电机质保期限（时间）                    |
| 电控质保期限（时间）     | controller_warranty_year   | 数值输入   | 否    | 正数，精度0.5，单位：年    | —       | 如5                                       |
| 电控质保期限（里程）     | controller_warranty_mileage | 数值输入   | 否    | 正整数，单位：公里        | —       | 如100000                                  |
| 电控质保到期日期       | controller_warranty_end    | 日期显示   | —    | 系统计算             | —       | 三电质保起始日期 + 电控质保期限（时间）                    |

**三电质保到期日期说明：**
- 电池、电机、电控各自独立计算到期日期
- 概览统计条中"三电质保到期日期"取三者中最早的日期
- 详情中分别展示各部件的到期日期，便于用户了解各部件的质保情况

**三电质保状态判断：** 分别对电池、电机、电控按整车质保状态相同的规则判断，取三者中状态最严重的作为三电质保整体状态（已过期 > 即将到期 > 质保期内）。

| 服务类型    | service_type           | 多选     | 否    | —    | —       | 枚举值：上门服务、到店服务、远程支持     |

#### 3.3.7 维保计划信息

| 字段名      | 字段标识                           | 字段类型   | 是否必填 | 校验规则             | 默认值 | 说明                                       |
| -------- | ------------------------------ | ------ | ---- | ---------------- | --- | ---------------------------------------- |
| 首保提醒日期  | first_maintenance_remind_date  | 日期选择   | 否    | —                | —   | 厂家建议的首保时间，如出厂后3个月                       |
| 首保提醒里程  | first_maintenance_remind_mileage | 数值输入   | 否    | 正整数，单位：公里        | —   | 厂家建议的首保里程，如1000公里                       |
| 首保日期    | first_maintenance_date         | 日期选择   | 否    | —                | —   | 实际完成首保的日期（事后记录）                         |

**维保计划说明：**
- 首保提醒日期/里程：厂家建议值，用于事前提醒，在首保未完成前展示
- 首保日期/里程：实际值，用于事后记录，首保完成后填写
- 保养周期（时间）和保养周期（里程）为"或"逻辑，先到为准

#### 3.3.8 下次保养提醒日期计算规则

| 条件                    | 计算方式                                    | 显示值       |
| --------------------- | --------------------------------------- | --------- |
| 首保未完成（首保日期为空）且首保提醒日期已填 | 直接使用首保提醒日期                              | 首保提醒日期    |
| 首保已完成且保养周期（时间）已填     | 首保日期 + 保养周期 × N月（N为已保养次数+1）            | 计算后的日期    |
| 上述条件均不满足              | 无法计算                                    | —         |

**说明：** 对接售后系统后，已保养次数从保养记录中自动获取，下次保养提醒日期实时更新。

### 4.6 编辑保存规则

1. 售后信息子Tab支持编辑，维修信息和保养信息子Tab为只读

3. 保存成功后自动记录变更日志（修改字段、修改前后值、操作人、操作时间等）
4. 系统计算字段（质保到期日期、质保状态、下次保养提醒日期等）不参与提交，由系统实时计算

## 5. 页面交互流程

### 5.1 售后维保Tab交互流程

```
用户进入资产详情页 → 切换至"售后维保"Tab
  → 显示概览统计条（始终可见）
  → 显示子Tab标签区：[售后信息] [维修信息(暂未开放)] [保养信息(暂未开放)]
  → 默认显示"售后信息"子Tab内容
  → 可选操作：
    ├── 查看质保信息、服务商信息、维保计划、附件
    ├── 点击"编辑" → 售后信息子Tab进入编辑态 → 修改 → 确定/取消
    ├── 切换至"维修信息"子Tab → 显示空态提示"维修记录将在对接售后系统后自动同步"
    └── 切换至"保养信息"子Tab → 显示空态提示"保养记录将在对接售后系统后自动同步"
```

### 5.2 编辑态交互流程

```
用户在售后维保Tab → 点击"编辑"
  → 售后信息子Tab进入编辑态
  → 可编辑字段变为可输入状态
  → 质保范围选择"无质保"时，质保起始日期和期限字段置灰
  → 质保起始日期/期限修改时，到期日期和质保状态实时计算
  → 点击"确定"：
    ├── 校验成功 → 保存数据，记录变更日志，退出编辑态，显示"保存成功"Toast
    └── 校验失败 → 在对应字段下方显示红色错误提示
  → 点击"取消" → 退出编辑态，不保存任何修改
```
